# Научная методика обработки термограмм

## 1. Входные данные

### 1.1. Колонки термограммы

| Колонка | Обязательность | Единицы | Описание |
|---------|---------------|---------|----------|
| temp | Обязательно | K (Кельвин) | Температура |
| deltatemp | Необязательно | °C (градусы Цельсия) | DTA-сигнал (разница температур) |
| time | Обязательно | мин (минуты) | Время эксперимента |
| mass | Обязательно | мг (миллиграммы) | Масса образца |

### 1.2. Автодетект разделителей

CSV: `,`, `;`, `\t`, пробелы. Заголовки нормализуются к именам колонок.
Файлы без заголовков: колонки по индексу (0=temp, 1=deltatemp, 2=time, 3=mass).

### 1.3. Валидация данных

- Все значения должны быть числовыми
- Пустые строки удаляются
- Строки с NaN в temp или time удаляются с предупреждением
- Начальные и конечные NaN НЕ экстраполируются
- Внутренние NaN в deltatemp/mass интерполируются линейно
- Доля удалённых и интерполированных строк сохраняется в metadata

## 2. Выравнивание нескольких термограмм

### 2.1. Физическая ось

Для стандартной обработки TGA используется температура как общая ось:

- `Tmin_common = max(Tmin каждого эксперимента)`
- `Tmax_common = min(Tmax каждого эксперимента)`
- `temperature_grid = linspace(Tmin_common, Tmax_common, bins)`

### 2.2. Интерполяция

Каждая термограмма интерполируется независимо на общую температурную сетку:

- Масса, DTA, время интерполируются отдельно
- Только валидные точки внутри диапазона конкретного эксперимента
- Экстраполяция запрещена
- Если общий диапазон отсутствует → `NoCommonRangeError`

### 2.3. Один файл

Для одной термограммы используется её собственный температурный диапазон и сетка из `bins` точек.

### 2.4. Текущая реализация (⚠️)

Текущий `resample_thermogram()` выравнивает по **нормализованному индексу строки** (0-1), а не по температуре. Это означает, что точки с одинаковым относительным номером ≠ одинаковая температура.

**Целевое решение**: интерполяция по общей температурной сетке.

## 3. Сглаживание

### 3.1. Метод

Обязательная зависимость: SciPy. Runtime branching удалён.

Savitzky-Golay filter: `scipy.signal.savgol_filter(data, window, polyorder)`

### 3.2. Параметры

- `window`: нечётное, > polyorder, ≤ длины непрерывного сегмента
- `polyorder`: 1-3 (обычно 2 или 3)
- `mode`: 'interp' (интерполяция краёв)

### 3.3. Правила

- Каждый непрерывный участок данных обрабатывается отдельно
- NaN-разрывы не соединяются
- Константа после сглаживания остаётся константой
- Линейная функция не искажается

### 3.4. Сравнение с R

| Аспект | R (tg.app) | Python (tg-py) |
|--------|-----------|---------------|
| Сглаживание DTG | `pspline::sm.spline` | SG (optional) + rolling mean |
| Сглаживание массы | Закомментировано | `rolling(window).mean()` |
| Сглаживание температуры | Закомментировано | `rolling(window).mean()` |
| Сглаживание DTA | Не применяется | SG через `sg_mode` |
| Сглаживание производной | `sm.spline` | spline-like через `span` |

## 4. Расчёт производной DTG

### 4.1. Правильный порядок

1. Валидация отдельного эксперимента
2. Выравнивание
3. Сглаживание массы отдельного эксперимента
4. Расчёт `dm/dt` отдельного эксперимента
5. Усреднение производных

### 4.2. Формула

```
dmdt[i] = (mass_smoothed[i + lag] - mass_smoothed[i]) / (time[i + lag] - time[i])
```

Или с `np.gradient`:

```
dmdt = np.gradient(mass_smoothed, time)
```

### 4.3. Требования перед расчётом

- Время должно быть строго возрастающим
- Единицы времени должны быть известны (минуты)
- Знак производной сохраняется и документируется (отрицательный = потеря массы)

### 4.4. Параметр difflag

Центральная разность с настраиваемым лагом. По умолчанию `lag=1`.

### 4.5. Текущая реализация (⚠️)

В `processing.py` производная `dmdt` рассчитывается **на mean frame** (усреднённом по всем термограммам), а не per-run. Это отличается от R, где `diff(mass, lag) / diff(time, lag)` применяется к каждому эксперименту отдельно.

Текущая реализация использует `np.interp` с нормализованным индексом (0-1) для приведения длины к `bins`, вместо интерполяции по температуре.

## 5. Температурная коррекция

### 5.1. Метод

Корректирующая кривая интерполируется по температуре, а не по относительному номеру строки.

1. Валидация correction-файла
2. Проверка температурного диапазона
3. Интерполяция `correction.deltatemp` на `temperature_grid`
4. Применение к каждой кривой или к средней кривой

### 5.2. Ошибки

Если коррекция не покрывает рабочий диапазон → понятная ошибка с фактическим и требуемым диапазонами.
Экстраполяция запрещена.

### 5.3. Текущая реализация (⚠️)

В `compute_mean_correction()` коррекция проходит через `resample_thermogram()`, который использует нормализованный индекс строки, а не температуру.

## 6. Скорость нагрева

### 6.1. Текущая (неправильная) формула

```
heating_rate = T_last / t_last
```

Предполагает `T_first = 0`, `t_first = 0`.

**R-исходник** (`server.r` строка 656):

```r
heat.speed <- tail(temp.mean,1) / tail(time.mean,1)
```

**Python-порт** (`summary.py` строка 63):

```python
return f"Скорость нагрева: {round(float(last['temp']) / time_value, 1):.1f} K/мин"
```

### 6.2. Правильная формула (целевая)

```
β = (T_last - T_first) / (t_last - t_first)
```

Или линейная регрессия: `T(t) = T0 + βt`, где β — коэффициент регрессии.

### 6.3. Дополнительные метрики

- β, K/мин
- R² коэффициента качества линейного приближения
- Минимальная и максимальная локальная скорость нагрева

## 7. Тепловой эффект

### 7.1. Алгоритм

1. Нормализовать границы xmin, xmax (не округлять до целых)
2. Отсортировать данные по температуре
3. Сдвиг DTA: `deltatemp_shifted = deltatemp - deltatemp[0]` — вычитаем значение первой точки, чтобы базовая линия начиналась с нуля
4. Интерполировать значения DTA точно на обеих границах
5. Добавить граничные точки в выбранный интервал
6. Построить линейную базовую линию между значениями сигнала на границах
7. Интегрировать: `integral(DTA(T) - baseline(T)) dT`
8. Применить калибровочный коэффициент EFFECT_SCALE_K
9. Разделить на начальную массу

### 7.2. Базовая линия

Прямая линия от первой до последней точки выбранного температурного интервала.

```
baseline(T) = DTA_first + (DTA_last - DTA_first) * (T - T_first) / (T_last - T_first)
```

### 7.3. Интегрирование

**R-исходник** (`server.r`):

```r
trapz(selection[,1], selection[,2]) - 0.5 * (tail(temp) - first(temp)) * tail(deltatemp)
```

**Python-порт** (`summary.py` строки 84-86):

```python
triangle_area = 0.5 * (float(last["temp"]) - float(first["temp"])) * float(last["deltatemp"])
trapz_area = float(np.trapezoid(shifted["deltatemp"].to_numpy(dtype=float), shifted["temp"].to_numpy(dtype=float)))
effect = trapz_area - triangle_area
```

**Паритет**: ⚠️ Требуется проверка эквивалентности сдвига DTA. R делает сдвиг `selection[,2] <- selection[,2] - selection[1,2]` перед интегрированием (server.r). Python делает то же самое `shifted["deltatemp"] -= shifted.iloc[0]["deltatemp"]` (summary.py). Оба реализуют одинаковый сдвиг, но в R это происходит внутри reactive context, в Python — явно в функции.

**R-исходник** (server.r, сдвиг):
```r
selection[,2] <- selection[,2] - selection[1,2]
```

**Python-порт** (summary.py, сдвиг):
```python
shifted["deltatemp"] = shifted["deltatemp"] - float(shifted.iloc[0]["deltatemp"])
```

### 7.4. Калибровочный коэффициент

**EFFECT_SCALE_K = 0.4458333**

- **Происхождение**: Калибровка по цинку (Zn), `../tg.app/server/server.r` строка 4: `K <- 0.4458333 # цинк`
- **Физический смысл**: Преобразование сырой площади DTA-сигнала в Дж/г
- **Вывод**: `K = ΔH_fusion_Zn / measured_area_per_gram_Zn ≈ 107.1 / 240.2 ≈ 0.4458`
- **Единицы**: (J/g) / (instrument_units²) — эмпирический коэффициент
- **Этап применения**: После интегрирования, перед делением на массу
- **Допустимость изменения**: Нет, калибровочный коэффициент. Изменение требует повторной калибровки

### 7.5. Единицы результата

```
effect = K * integral(DTA(T) - baseline(T)) dT / mass
```

Единицы: **(J/g)** — джоули на грамм образца.

### 7.6. Текущая реализация (⚠️)

В `build_effect_text()` границы `xmin`/`xmax` округляются до целых чисел (`round(float(xmin), 0)`), что может потерять точность. Правильно: нормализовать границы без округления.

## 8. Поиск пиков

### 8.1. Метод

Текущая реализация использует кастомный алгоритм поиска локальных экстремумов с окном и prominence-порогом, основанным на MAD (median absolute deviation).

**R-исходник**: `stat_peaks` / `stat_valleys` из ggplot2 (через `stat_summary`).

**Python-порт** (`peaks.py`):

- `_detect_extrema_points()`: скользящее окно, сравнение с локальным минимумом/максимумом
- `_estimate_noise_sigma()`: MAD-based оценка шума
- `detect_peaks()`: поиск пиков на DTA и DTG

### 8.2. Параметры

- `prominence`: минимальная высота пика относительно окрестности
- `distance`: минимальное расстояние по температуре между пиками
- `width`: минимальная ширина пика
- `height`: минимальная высота пика (опционально)

### 8.3. Шум

Noise sigma оценивается через MAD (median absolute deviation) от rolling mean residual.

```python
mad = float(np.median(np.abs(residual - np.median(residual))))
noise_sigma = 1.4826 * mad  # константа 1.4826 для нормального распределения
```

Минимальная `prominence = prominence_sigma * noise_sigma`, `prominence_sigma` по умолчанию **5.0**.

### 8.4. Текущая реализация (⚠️)

Не используется `scipy.signal.find_peaks`. Кастомный алгоритм с фиксированным окном может давать другие результаты, чем `find_peaks` с параметрами `prominence`, `distance`, `width`.

## 10. Предупреждения и ошибки

### 10.1. Предупреждения (non-fatal)

- Удалены N строк с NaN в temp/time
- Интерполировано N точек в deltatemp/mass
- Сужение общего диапазона при усреднении нескольких файлов

### 10.2. Фатальные ошибки

- Недостаточно точек после очистки (< минимальное)
- Нет общего температурного диапазона
- Correction-файл не покрывает рабочий диапазон
- Начальная масса ≤ 0
- Время немонотонно (существенное обратное движение)
- Пустой файл

## 14. Пайплайн обработки

### 14.1. Целевой пайплайн

```
Загрузка файлов
    │
    ├─→ Автодетект разделителя
    ├─→ Нормализация колонок
    └─→ Валидация (NaN, типы, монотонность)
         │
         ├─→ Для каждой термограммы:
         │    ├─→ Выравнивание по общей температурной сетке ⚠️ (текущий: по индексу строки)
         │    ├─→ Сглаживание массы (SG, scipy)
         │    └─→ Расчёт dm/dt per-run (np.gradient) ⚠️ (текущий: на mean frame)
         │
         ├─→ Усреднение производных (не масс)
         ├─→ Температурная коррекция по температуре ⚠️ (текущий: по индексу строки)
         │
         ├─→ Сглаживание DTA (SG через sg_mode)
         ├─→ Расчёт скорости нагрева ⚠️ (текущий: T_last/t_last)
         ├─→ Поиск пиков (scipy.signal.find_peaks) ⚠️ (текущий: кастомный window-based)
         └─→ Расчёт теплового эффекта
              ├─→ Выбор температурного интервала
              ├─→ Сдвиг DTA: deltatemp - deltatemp[0]
              ├─→ Базовая линия (прямая first→last)
              ├─→ Интегрирование (trapz - triangle)
              ├─→ K * effect / mass
              └─→ Результат: Дж/г
```
